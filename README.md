#  Manual Testing Portfolio – SauceDemo Login Page

## 📌 Overview

This project demonstrates my manual testing skills through testing the **Login functionality** of the SauceDemo web application.

The goal of this testing activity is to validate system behavior, identify defects, and evaluate overall usability and reliability from an end-user perspective.

---

## 📅 Project Information

* **Application:** SauceDemo
* **Module:** Login Page
* **Test Type:** Manual Testing
* **Test Period:** March 29, 2026 – March 30, 2026
* **Environment:**

  * Desktop: Chrome (v146)
  * Mobile: Safari

---

## 🎯 Scope of Testing

* Login functionality validation
* Input validation (valid/invalid scenarios)
* Error handling and messaging
* UI/UX verification
* Edge cases and negative scenarios

---

## 🧪 Testing Activities

* Analyzed system behavior and defined test scenarios
* Designed and executed detailed test cases
* Applied risk-based and exploratory testing
* Identified UI/UX inconsistencies and validation issues
* Reported and documented defects with clear steps and evidence

---

## 📊 Test Deliverables

### ✅ Test Cases

* Designed comprehensive test cases covering:

  * Positive scenarios (valid login)
  * Negative scenarios (invalid credentials, empty fields)
  * UI/UX validation

### 🐞 Bug Reports

Identified and reported multiple defects, including:

* Missing error message for blocked user under specific conditions
* Truncated and incorrectly formatted error message
* Redundant UI elements (duplicate close icons in input fields)

## ⚠️ Risks Identified

* Unlimited login attempts may lead to brute-force attacks, allowing unauthorized access to user accounts

---

## 📌 Key Findings

* Core login functionality works as expected under normal conditions
* Several UI/UX and validation issues were identified
* Some issues may impact user experience but do not block core functionality

---

## ✅ Conclusion

The login feature is generally stable and functional. However, improvements are needed in error handling, UI consistency, and validation behavior to enhance user experience.

Fixing the identified defects and performing regression testing is recommended to ensure system stability and quality.
